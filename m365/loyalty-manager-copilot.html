How It Works

A simple Copilot flow that feels operational, not abstract

1. Choose the issue to fix The user starts with plain-language issue types instead of technical backend categories.
2. Answer a few workflow questions The agent asks about data sources, sample-data fallback, approval behavior, and goodwill handling.
3. Review the proposed action The agent presents what it found and what it would do as an operator.
4. Approve simulated execution The workflow asks for approval where appropriate and then simulates the fix.
5. See before and after results The user gets a synthesized view of what changed so the workflow feels concrete and credible.

Installation & Permissions

Available in the Microsoft Teams Store

Loyalty Manager Copilot AI is published by IntelliThing, Inc. and available in the Microsoft Teams App Store and Microsoft 365 App Catalog.

IT admin note: As with all Microsoft 365 Copilot agents, your organization's IT admin may need to approve this app before users can add it. When a user clicks Request approval, the request appears in the M365 Admin Center under Agents → All agents → Requests.

Permissions requested: identity (identifies the signed-in user) and messageTeamMembers (allows the agent to send response cards in Teams). No Microsoft Graph write permissions are requested. The agent cannot read email, calendar, files, or contacts.

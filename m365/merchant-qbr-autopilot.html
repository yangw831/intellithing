Installation

Requires IT admin approval — takes minutes

Like all Microsoft 365 Copilot agents from third-party publishers, Merchant QBR AutoPilot requires approval from your IT admin before it can be added. This is standard M365 tenant policy.

Here's the fastest path:

  1. Find the app on the Microsoft Teams store or Marketplace and click Request approval.
  2. Forward the IT admin guide to your IT team.
  3. Your admin approves it at M365 Admin Center → Agents → All agents → Requests.
IT Admin Approval Guide

Permissions requested

  • User identity — to personalize responses
  • Message team members — to send response cards in Teams
  • Backend API access — read-only access to merchant scorecard data

No Microsoft Graph write permissions. The agent cannot read your email, calendar, or files. No user data is stored outside Microsoft's Azure infrastructure.

FAQ

Common questions

Is this app free?

Yes. Merchant QBR AutoPilot is free to use. No subscription or purchase required.

Do I need a separate account?

No. Users authenticate with their existing Microsoft 365 work account. No sign-up required.

Can I use my own merchant data?

Yes. Upload a CSV following the published schema and the agent generates a scorecard from your data. Sample merchants are also available for immediate use without any upload.

Where is data processed and stored?

All processing happens in IntelliThing's Azure-hosted backend. No user data is stored or transmitted outside of Microsoft's Azure infrastructure. Generated files are stored in Azure Blob Storage with time-limited download links.

What Microsoft 365 plan do I need?

The app requires a Microsoft 365 plan that includes access to Microsoft 365 Copilot and Teams.

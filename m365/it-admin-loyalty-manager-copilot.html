What it does

Loyalty Manager Copilot AI is a Copilot agent that helps loyalty managers, CRM teams, and operations staff diagnose loyalty program issues, analyze program performance, and prepare executive-ready reporting — all from a guided conversation inside Microsoft 365 Copilot. No separate backend account or subscription is required.

Users authenticate with their existing Microsoft 365 identity. The agent uses sample data when live integrations are not configured and is transparent about this at every step.

Who uses it

Loyalty program managers, CRM and retention leads, Shopify store owners, operations teams, and marketing leaders who work within Microsoft 365.

Permissions requested and why

Loyalty Manager Copilot AI has a base Copilot agent approval path and an optional report-workspace save path. Report generation and download can be tested without approving file write access. Saving a generated report into Microsoft 365 requires a one-time tenant admin consent step.

PermissionTypeWhy it's needed
identity Microsoft 365 standard Identifies the signed-in user to personalize responses
messageTeamMembers Microsoft 365 standard Allows the agent to send response cards within Teams conversations
access_as_user Delegated backend API scope Allows the report workspace action to call the Loyalty Manager Copilot AI backend as the signed-in user
Files.ReadWrite Microsoft Graph delegated scope Allows the backend to save a generated PowerPoint report to the signed-in user's Microsoft 365 report workspace
Files.ReadWrite.AppFolder Microsoft Graph delegated scope Supports the app-folder workspace under My files / Apps / Loyalty Manager Copilot AI / Reports
offline_access Microsoft identity delegated scope Allows Microsoft identity to maintain the consented session for the approved app permissions
Chat.Create, ChatMessage.Send, User.ReadBasic.All Microsoft Graph delegated scopes Allow the optional Teams sharing action to resolve an internal recipient, create a one-to-one chat, and send the report card

The app does not request permission to read email, calendar, contacts, or all SharePoint sites. The report-save backend writes only the generated report file selected by the user into the app's report workspace folder.

Where data lives

  • Sample-data workflows can be tested without connecting customer systems.
  • Generated reports are temporarily stored by the Loyalty Manager Copilot AI Azure service to provide a download link and optional Microsoft 365 save action.
  • If workspace save is approved, reports are saved to the signed-in user's OneDrive app folder: My files / Apps / Loyalty Manager Copilot AI / Reports.
  • Authentication is handled by Microsoft Entra ID. IntelliThing never sees user passwords.

How to approve the app installation

Loyalty Manager Copilot AI is published in the Microsoft commercial marketplace, so people in your organization can find it in Microsoft 365 Copilot, Teams, and the Microsoft 365 app. Depending on your tenant's app policies, a user may need administrator approval before they can install and use it.

For app users — how to request the app

If you don't have permission to install apps yourself, send an approval request to your administrators:

  1. In Microsoft 365 Copilot, Teams, or the Microsoft 365 app, search for Loyalty Manager Copilot AI by IntelliThing, Inc.
  2. Open the app and select Add (or Get it now).
  3. If your organization requires approval, you'll see a Request approval option instead. Select it, add a short justification, and submit.

Your request is then sent to your Microsoft 365 administrators, who approve it as described below. The exact button and screen can vary by tenant policy and Microsoft 365 client.

For IT admins — how to approve a request

Step 1: Sign in to admin.cloud.microsoft

Step 2: Navigate to either:

  • Agents → All agents → Requests (recommended for Copilot agents)
    https://admin.cloud.microsoft/#/agents/all/requested
  • Settings → Integrated apps → Requested apps (also works)
    https://admin.cloud.microsoft/#/Settings/IntegratedApps

Step 3: Find Loyalty Manager Copilot AI by IntelliThing, Inc. and select Accept request.

Once approved, the app becomes available to the requesting user. You can also deploy it to a broader group from the same screen.

Don't see the request in the queue? An empty Requests list is normal and usually means one of these:
  • No request has been submitted yet. The Requests queue only shows apps a user has actively requested. Ask the requesting user to follow the "For app users" steps above so their request appears. Alternatively, you can install it for them proactively: find Loyalty Manager Copilot AI under Settings → Integrated apps → Available apps and deploy it to the user or a group.
  • The request was already handled. If it was previously approved, the app is already available to the user — check Integrated apps → Deployed apps (or Agents → All agents). If it was rejected, the user can submit a new request, or you can deploy it directly from Available apps.

How to approve report workspace save

Loyalty Manager Copilot AI lets a user save a generated loyalty performance report as a PowerPoint file to their own OneDrive, in an app-specific folder (My files / Apps / Loyalty Manager Copilot AI / Reports). Because the report is written on the user's behalf, this feature requires a one-time tenant admin consent for the Microsoft Graph file permissions and the app's backend API scope. Open the admin consent link below, review the requested permissions, and approve on behalf of your organization.

Grant report workspace consent

After approval, verify in Microsoft Entra admin center under Enterprise applications → Loyalty Manager Copilot AI → Permissions that the tenant has grants for Files.ReadWrite, Files.ReadWrite.AppFolder, offline_access, and access_as_user.

If the Enterprise Application already exists and shows only offline_access, delete the stale Enterprise Application entry or revoke the stale delegated grant, then open the admin consent URL again.

How to approve report sharing in Teams

Loyalty Manager Copilot AI lets a user share a saved report with a colleague as a Microsoft Teams chat message, so the recipient gets the report card and file directly in Teams. Because the message is sent on the user's behalf, this feature requires a one-time tenant admin consent for the Microsoft Graph Teams chat and message permissions, the file permission used to attach the report, and the app's backend API scope. It uses a separate application from workspace save because of these additional Teams permissions. Open the admin consent link below, review the requested permissions, and approve on behalf of your organization.

Grant report sharing consent

After approval, verify the report sharing Enterprise Application has grants for Files.ReadWrite, Chat.Create, ChatMessage.Send, User.ReadBasic.All, and access_as_user.

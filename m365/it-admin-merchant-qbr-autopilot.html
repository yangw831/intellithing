A user in your organization has requested this app. This page gives you everything you need to evaluate and approve it quickly.

Where users find it

Users discover Merchant QBR AutoPilot in the Microsoft Teams app store, Microsoft Marketplace, or the Microsoft 365 Copilot agent catalog by searching for Merchant QBR AutoPilot or merchant qbr. If tenant policy blocks self-service installation, they see a Request approval option instead of immediate access.

What it does

Merchant QBR AutoPilot is a Copilot agent that helps account managers generate Quarterly Business Review (QBR) scorecards directly inside Microsoft 365 Copilot and Teams. Users select a merchant, provide period data, and the agent returns a structured scorecard with an executive summary and a downloadable PowerPoint deck — eliminating manual report preparation.

No separate account or subscription is needed. Users authenticate with their existing Microsoft 365 identity.

Who uses it

Account managers, merchant success managers, and business operations teams who prepare recurring QBR readouts for merchants or partners.

Permissions requested and why

PermissionTypeWhy it's needed
identity Microsoft 365 standard Identifies the signed-in user to personalize responses
messageTeamMembers Microsoft 365 standard Allows the agent to send response cards within Teams conversations
Backend API access (custom) Delegated, user-consented Allows Copilot to call IntelliThing's merchant data API on behalf of the signed-in user to retrieve scorecard inputs. Read-only. No access to Microsoft 365 data.

No Microsoft Graph write permissions are requested. The agent cannot read email, calendar, files, or contacts. It cannot write to any Microsoft 365 service.

Where data lives

  • All merchant data processing occurs in IntelliThing's Azure-hosted backend (Azure Functions, Azure Blob Storage).
  • No user data is stored or transmitted outside of Microsoft's Azure infrastructure. IntelliThing's backend runs on Azure.
  • CSV data uploaded by users is processed in memory for scorecard generation and is not persisted.
  • Generated PowerPoint artifacts are stored in Azure Blob Storage with time-limited download links.
  • Authentication is handled entirely by Microsoft Entra ID (OAuth 2.0 delegated flow). IntelliThing never sees user passwords or raw tokens.

How to approve

When a user clicks Request approval on the app listing, the request appears in the M365 Admin Center.

Step 1: Sign in to admin.cloud.microsoft

Step 2: Navigate to either:

  • Agents → All agents → Requests (recommended for Copilot agents)
    https://admin.cloud.microsoft/#/agents/all/requested
  • Settings → Integrated apps → Requested apps (also works)
    https://admin.cloud.microsoft/#/Settings/IntegratedApps

Step 3: Find Merchant QBR AutoPilot by IntelliThing, Inc. and select Accept request.

Once approved, the app becomes available to the requesting user. You can also deploy it to a broader group from the same screen.

If no request is visible yet

Ask the user to search for Merchant QBR AutoPilot in Teams Apps or Microsoft 365 Copilot and click Request approval. You can also look it up from Microsoft Marketplace using the product name and then manage availability from your tenant's app and agent controls.
